Puerto 0x60/0x64 IRQ 1 Esperando...

💡 ¿Cómo funciona el Controlador de Teclado? Aprende sobre el i8042

El controlador i8042 es el chip que gestiona el teclado y el mouse en la computadora.

  • ⌨️ Scan Codes: Cada tecla genera un código único (scan code) cuando la presionas
  • 📍 Puertos E/S: Usa el puerto 0x60 para datos y 0x64 para comandos/estado
  • ⚡ IRQ 1: Genera una interrupción cada vez que presionas una tecla
  • 🔄 Buffer: Almacena hasta 16 bytes de datos de teclas presionadas
Ejemplo: Cuando presionas la tecla 'A':
  1. El teclado envía el scan code 0x1E al puerto 0x60
  2. El controlador activa IRQ 1 para avisar a la CPU
  3. La CPU lee el código desde el puerto 0x60
  4. El sistema operativo convierte 0x1E en la letra 'A'

Puerto 0x60/0x64 IRQ 12 Esperando...

💡 ¿Cómo funciona el Controlador de Mouse? Protocolo PS/2

El mouse PS/2 envía paquetes de 3-4 bytes cada vez que lo mueves o haces clic.

  • 📦 Paquetes de datos: Cada movimiento genera 3 bytes (estado, X, Y)
  • 🖱️ Movimiento relativo: Reporta cuánto se movió, no la posición absoluta
  • ⚡ IRQ 12: Genera una interrupción cuando hay datos disponibles
  • 🔢 Resolución: Típicamente 200-800 DPI (puntos por pulgada)
Paquete de datos del mouse:
  • Byte 1: Estado de botones (bits para izquierdo, derecho, medio)
  • Byte 2: Movimiento en eje X (-127 a +127)
  • Byte 3: Movimiento en eje Y (-127 a +127)
  • Byte 4: Scroll wheel (si está presente)

🔄 CONTROLADOR DMA (Direct Memory Access)

Canales: 8 DMA1: 0x00-0x0F DMA2: 0xC0-0xDF Activo

💡 ¿Qué es DMA? Acceso Directo a Memoria

DMA permite que los dispositivos transfieran datos directamente a la memoria sin usar la CPU.

  • 📦 Sin CPU: Las transferencias ocurren sin intervención del procesador
  • ⚡ Más rápido: Libera a la CPU para otras tareas mientras se transfieren datos
  • 🔢 8 Canales: Cada dispositivo puede usar un canal DMA exclusivo
  • 💾 Memoria directa: Los datos van directamente del dispositivo a la RAM
Ejemplo: Cuando lees un archivo del disco, el controlador DMA transfiere los datos directamente del disco a la memoria RAM sin que la CPU tenga que copiar cada byte.

💡 ¿Qué son las Interrupciones (IRQs)? Señales de los dispositivos

Las interrupciones son señales que los dispositivos envían a la CPU para avisar que necesitan atención.

  • ⚡ Señal urgente: El dispositivo interrumpe lo que la CPU está haciendo
  • 🔢 IRQ Number: Cada dispositivo tiene un número de IRQ único (0-15 en PC)
  • ⏱️ Respuesta rápida: La CPU detiene su tarea actual y atiende la interrupción
  • 🔄 Rutina de servicio: Se ejecuta código específico para manejar el dispositivo
Ejemplo de IRQs comunes:
  • IRQ 0: Temporizador del sistema (ticks cada 1ms)
  • IRQ 1: Teclado (cada vez que presionas una tecla)
  • IRQ 12: Mouse PS/2 (cada movimiento o clic)
  • IRQ 14/15: Controladores de disco (IDE)

¿Por qué son importantes? Sin interrupciones, la CPU tendría que estar constantemente verificando cada dispositivo para ver si tiene datos, desperdiciando mucho tiempo.
